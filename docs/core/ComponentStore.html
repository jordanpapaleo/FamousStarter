ComponentStore

ComponentStore [constructor] 

ComponentStore manages components and renderables. It also keeps track of the size shared by all renderables managed by this ComponentStore.

Every LocalDispatch has its own ComponentStore.

new ComponentStore()

Instance methods

.clearComponents()

Clears all components by delegating to the layer they are being managed on.

.clearRenderables()

Clears all renderables by delegating to the layer they are being managed on.

.clear()

Clears all components and renderables managed by this ComponentStore by delegating to the respective layers.

.cleanComponents()

Cleans the underlying layer responsible for maintaining components.

.cleanRenderables()

Cleans the underlying layer responsible for maintaining renderables.

.clean()

Cleans the renderables and components managed by this ComponentStore.

.requestComponentId()

Returns a new component id that can be used in order to register a new component on the ComponentStore using registerComponentAt.

.requestComponentId()

Returns a new renderable id that can be used in order to register a new renderable on the ComponentStore using registerRenderableAt.
