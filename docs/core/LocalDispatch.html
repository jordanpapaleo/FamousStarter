LocalDispatch

LocalDispatch [constructor] 

As opposed to a Node, a LocalDispatch does not define hierarchical structures within the scene graph. Thus removing the need to manage children, but at the same time requiring the Node to delegate updates to its own LocalDispatch and all subsequent Nodes.

The primary responsibilty of the LocalDispatch is to provide the ability to register events on a specific Node ("targeted events"), without inducing the complexity of determining the Nodes location within the scene graph.

It also holds a reference to a RenderContext, therefore being required to delegate invocations of its update function to its RenderContext, which consequently mutates the actual 3D transform matrix associated with the Node.

Triggers an event on the Node attached to the LocalDispatch. Events are being managed by the GlobalDispatch.

Parameters:

NameTypeDescription

event type to listen on

event payload object to be passed in to every callback function attached to the specified event type

Instance methods

.kill()

Kills the componentstore of the LocalDispatchm therefore killing all Renderables and Components registered for the managed node.

.getNode()

Returns the managed Node.

.getContext()

Retrieves the RenderContext managed by the LocalDispatch.

.getRenderPath()

Returns the RenderPath uniquely identifiying the Node managed by the LocalDispatch in the scene graph.

.getRenderProxy()

Returns the RenderProxy managed by the LocalDispatch.

.registerTargetedEvent(event, cb)

Registers an event listener to be triggered whenever the specified event is being triggered on the path defined by the RenderProxy attached to the LocalDispatch describing the Node's location in the Scene graph.

NameTypeDescription

.registerGlobalEvent(event, cb)
